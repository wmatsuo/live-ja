Brightcoveは、すべての Brightcove サービスとアプリケーションのステータスを継続的に監視します。サービスが中断された場合は、システムステータスページにメモが掲示されます。本ページには、予定されているメンテナンス通知もすべて掲載されます。

システムステータスページにアクセスするには、 https://status.brightcove.comにアクセスするか、ナビゲーションヘッダーから [サポート] アイコン ()、 システムステータス。

投稿されたすべての通知には、以下の情報が含まれます。

  • インシデントステータス
    • 運用可能 (緑)
    • 通知 (青-システムは現在動作中ですが、遅延または断続的な停止の影響を受ける)
    • 部分的なサービス中断 (黄色)
    • フルサービスの中断 (赤)
  • コンポーネント-サービスまたは影響を受ける領域
  • 所在地
    • APAC
    • 南北アメリカ
    • エメア
    • 日本
  • 現在の状態とステータス。州には以下のものがある。
    • 調査中-エンジニアが問題を調査中
    • 特定-エンジニアが問題を特定し、修正を検討している
    • モニタリング-問題の解決とエンジニアリング監視により、すべてが正常であることを確認する
    • 解決済み-問題が解決しました

投稿されたメッセージはすべて英語で、使用時間は東部標準時 (GMT-5) です。

システムステータス履歴の表示

システムステータスの履歴を表示するには、ページ上部の[履歴]リンクをクリックします。

システム状態の変更を購読する

サインアップして、システムステータスが変更されたときに通知を受け取ることができます。このセクションの構成は次のとおりです。

  • メールを購読する
  • Webhook を購読する
  • サブスクリプションを管理する

メールを購読中

電子メールで通知を受信するには、[ 購読]ボタンをクリックし、メールアドレスを入力し、[ 購読] をクリックします。

Webhook を購読する

ウェブフックは、HTTP POSTで作成された定義されたコールバックです。ステータスの更新がある場合に、コールバックが JSON オブジェクトをポストする URL を指定します。

システムステータスが変更されたときに通知を受け取るようにサインアップするには、[ SUBSCRIBE]ボタンを選択します。次に、電子メールアドレスとともに JSON ペイロードを受信する URL を入力します。

システムステータスの変更が送信されると、Webhook アドレスは次のような JSON オブジェクトを受け取ります。

サブスクリプションの管理

購読が完了すると、サブスクリプションを管理するための [サブスクリプションの管理] ページへのリンクが記載されたメールが届きます。通知の受信を希望するコンポーネント/製品/リージョンを選択できます。項目を選択し、[購読の保存]をクリックします。

購読を解除するには、[サブスクリプションの管理] ページの下部にある [ 登録解除]リンクをクリックします。

Slack で通知を取得する

Slack には独自の Webhook システムがあり、ステータス更新を接続できます。詳細については、 Slack のドキュメントを参照してください。
